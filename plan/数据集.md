# 用 GnesDA 适配 Alpha Satellite Monomer Embedding 的可行性分析

## Summary

  可以实现，但前提是按该仓库的原始范式做“有监督适配”，不能把它当成现成的零样本 DNA embedding
  模型直接用。

  你的目标可拆成两层：

- 生成每个 monomer 的 embedding：这点可以，模型本身就会输出单条序列 embedding。
- 让两个 monomer 的 embedding 距离和序列编辑距离强相关：这点也有希望，因为当前损失函数包含
  显式的距离拟合项，不只是近邻排序。

  但要注意，这个仓库的训练监督来自“真实距离矩阵”，不是自监督。也就是说，必须先对一批 monomer
  计算真实编辑距离，再训练模型去拟合它。

## Key Changes

- 数据输入改造
  现有代码只接受 data/<dataset>/train_seq_list、query_seq_list、base_seq_list 这类 pickle
  序列列表；你当前已有的是 FASTA 文件 inputs/monomers_train_all.fa 和 inputs/
  monomers_eval_all.fa。需要增加一个 FASTA -> 该目录结构的预处理步骤。
- 距离定义锁定为 ed
  你的目标明确要求和“编辑距离”强相关，因此应使用 --data_type dna --dist_type ed。不要用
  nw，因为当前 nw 分支返回的是比对百分比 pidentity，语义不是编辑距离，且和损失中的“距离越小
  越近”方向并不天然一致。
- 训练集构造
  需要把 monomer 集合拆成 train / query / base 三部分，并由代码离线计算：
    - train 内部两两编辑距离
    - query 到 base 的编辑距离
      这是当前训练与评估流程的核心监督来源。
  已有保存逻辑，会产出 embedding_xt.npy、embedding_xq.npy、embedding_xb.npy。
  当前仓库主要看 retrieval recall，不直接衡量“embedding 距离与编辑距离的相关性”。需要补一个
  评估脚本，至少计算：
    - Pearson 相关
    - Spearman 相关
    - embedding 距离 vs 编辑距离散点图/分箱统计
      否则无法回答“强相关”是否真的达成。

## Feasibility Judgment

- 在“纯 A/C/G/T”前提下，现有 DNA 分支可直接编码你的 monomer，字符集层面没有障碍。
- 从训练目标看，当前 TripletLoss 里有显式 mse_loss，会逼近真实距离，这比单纯 metric
  learning 更接近你的目标。
- 对 alpha satellite monomer 这类长度相近、字母表很小的 DNA 序列，任务本身是合理的，理论上
  比复杂蛋白序列更容易学到和编辑距离一致的几何结构。
- 最大风险不在模型结构，而在数据量与监督覆盖：
    - 如果 monomer 数量太少，模型容易过拟合，相关性未必稳定。
    - 如果只在很窄的距离范围训练，embedding 可能只学到局部排序，不一定全局线性相关。
    - 当前网络有较强下采样和 patching，若 monomer 很短，需检查 conv_layers / patch_len /
      stride 是否过重，否则会损失细粒度差异。

## Test Plan

- 小规模验证
  从 monomer 中先抽一个可控子集，计算全对编辑距离，训练一次 dist_type=ed。
- 直接看相关性
  在 held-out query/base 对上，计算 embedding 距离与真实编辑距离的 Pearson/Spearman。
- 看检索一致性
  保留现有 recall 评估，确认 embedding 近邻和编辑距离近邻基本一致。
- 做超参敏感性检查
  至少比较 conv_layers、patch_len、stride 对相关性的影响，尤其当 monomer 较短时。
- 做基线对比
  和简单 one-hot k-mer 统计或直接编辑距离 KNN 的表现做对照，确认 GnesDA 的 embedding 不
  是“看起来复杂但没有额外收益”。

## Assumptions

- monomer 序列仅含 A/C/G/T。
- 你接受先离线计算一批 monomer 的真实编辑距离作为监督。
- 目标是“学到和编辑距离强相关的 embedding”，不是“无需训练直接泛化到任意 DNA 序列”。
- 默认使用 ed，不使用 nw 作为主目标距离。
